1.启动未加载
	实例启动，oracle分析数据库参数文件，并通过设法获得实例锁来确定实例还未运行，启动多种不通数据库进程，例如dbwn和lgwr进程，oracle还将分配SGA所需的内存。




2.数据库加载未启动
	启动进程师第二个阶段数据加载，打开控制文件，获取数据库文件名，打开数据库数据文件做准备，此时不会检测数据文件是否存在，只是从控制文件中标识数据文件。

5.RMAN组成部分
	可执行文件
	recover.bsq：库文件
	协调信道访问sys数据包:
		dbms_rcvman:
			访问控制文件中的表并将信息传递给RMAN，执行操作前设置TIME运算符和效验数据文件中的检查点信息，并检查文件位置和大小。
		DBMS_BACKUP_RESTORE:
			访问控制文件并效验所需必须的信息，之后由于recover.bsq代码创建PL/SQL块，创建系统调用备份数据文件,控制文件，归档重做日志

	数据库权限：需要访问sys模式存在的各种数据包（sys.dbms_rcvman）
	数据库控制文件：负责数据库物理结构，查找文件位置，控制每个文件包含的头信息，RMAN使用控制文件编译文件列表，得到检查点信息，并确定可恢复性，直接访问控制文件，CONTROFILE_FILE_RECORDD_KEEP_TIME参数控制，重新构造控制文件
		alter database backup control file to trace；
		查看控制文件:
		select name from v$controlfile;
		快照控制文件：存储数据库结构信息，包括恢复检查点SCN信息，连续的SCN
数据块备份概述
	基于算法规则编译要备份的文件列表，基于信道和同时备份的文件数，RMAN再oracle共享段创建一些内存缓冲区
